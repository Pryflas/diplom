
# Прогнозирование риска заболеваний на основе нейросетевых архитектур и интерпретируемости

Ключевые слова: прогнозирование риска заболеваний, нейронные сети, многомерные клинические данные, интерпретируемость машинного обучения, SHAP

## Аннотация

Разработана архитектура нейросетевой системы для прогнозирования риска заболеваний на основе клинических данных пациентов. На основе сравнительного анализа пяти основных парадигм машинного обучения (ансамбли бустинга, полносвязные нейронные сети, рекуррентные сети, гибридные системы с методами объяснения, трансформер-подобные архитектуры) были идентифицированы ключевые критерии для достижения целевого показателя точности AUC-ROC ≥ 0.90. Предложена гибридная архитектура, объединяющая глубокую нейронную сеть (DNN) с методами SHAP для интерпретируемости, которая позволяет достичь высокой предсказательной способности при обработке табличных медицинских данных с дисбалансом классов и пропущенными значениями. Архитектура включает модули предварительной обработки данных (SMOTE, нормализация, кодирование признаков), трёхслойную нейронную сеть с регуляризацией (Dropout, Batch Normalization) и систему объяснения прогнозов через SHAP-значения. Результаты анализа показывают, что предложенный подход обеспечивает как высокую точность, так и клиническую интерпретируемость результатов, необходимую для внедрения в системы поддержки принятия врачебных решений.

## Введение

Своевременное и точное прогнозирование риска заболеваний является критической задачей современной системы здравоохранения, позволяя организовать эффективные профилактические мероприятия и оптимизировать использование ресурсов. Традиционные методы оценки риска (логистическая регрессия, шкалы оценки типа SCORE) часто ограничены в способности выявлять сложные нелинейные взаимодействия между множеством клинических факторов. Современные методы на основе нейронных сетей демонстрируют значительный потенциал в преодолении этих ограничений. Например, для задачи прогнозирования коронарной болезни сердца специализированные нейросетевые архитектуры увеличивают AUC-ROC с 0.806 до 0.942, что соответствует приросту более чем на 16.9% [4]. Аналогично, модели на основе искусственного интеллекта демонстрируют рост AUC до 0.818 против 0.732 у стандартной шкалы SCORE для оценки сердечно-сосудистого риска, обеспечивая прирост в 11.8% [1]. Такая точность достигается за счёт способности нейронных сетей автоматически выявлять скрытые паттерны во множестве разнородных признаков — от демографических характеристик и результатов лабораторных тестов до генетических маркеров. Однако одной из ключевых проблем при внедрении таких систем в клиническую практику является интерпретируемость получаемых прогнозов. Врачи и регуляторные органы требуют понимания того, на основе каких конкретных клинических факторов модель выносит свой вердикт. Это делает актуальной разработку гибридных подходов, которые сочетают высокую предсказательную точность с объяснимостью решений.

Описание проблемы: Ограниченная точность традиционных методов и недостаточная интерпретируемость современных методов машинного обучения при прогнозировании риска заболеваний затрудняют их внедрение в практику здравоохранения.

Актуальность исследования: Раннее и точное прогнозирование риска заболеваний позволяет своевременно организовать профилактические мероприятия. Современные методы на основе нейронных сетей демонстрируют высокую точность и адаптивность при работе с многомерными медицинскими данными. Нейросетевые подходы показывают потенциал в повышении точности прогнозирования и требуют внедрения в системы поддержки принятия врачебных решений.

Объект исследования: Нейросетевая модель прогнозирования риска заболеваний на основе клинических данных пациентов.

Предмет исследования:* Уровень точности и интерпретируемость прогнозирования риска заболевания, достигаемые с помощью нейросетевых моделей при обработке табличных медицинских данных с дисбалансом классов.

Цель дипломного исследования: Разработать нейросетевую модель для прогнозирования риска заболеваний на основе клинических данных, обеспечивающую AUC-ROC не менее 90% с гарантией интерпретируемости результатов.

Цель на текущий семестр: Спроектировать архитектуру нейронной сети для многомерных медицинских данных, провести сравнительный анализ альтернативных подходов и выбрать подходящий метод решения.

Задачи текущего этапа:

1. Сравнить и обосновать выбор архитектуры нейронной сети для анализа многомерных клинических данных путём систематического сравнения основных парадигм машинного обучения [2, 3].
2. Провести сравнительный анализ эффективности различных подходов (ансамбли бустинга, DNN, LSTM, гибридные системы, трансформеры) по комплексу объективных критериев (прогностическая точность, обработка табличных данных, интерпретируемость) для выявления подходящей конфигурации [1, 4].
3. Обосновать выбор метода решения на основе проведённого анализа и спроектировать архитектуру системы, включающую модули предварительной обработки, нейросетевого классификатора и интерпретации результатов [2, 4].
4. Выявить и интерпретировать ключевые признаки, влияющие на прогноз, используя методы SHAP и LIME, для валидации результатов с медицинской точки зрения.

Выбор этих аналогов обусловлен необходимостью анализа различных подходов к обработке многомерных клинических данных, их способности достичь целевого показателя AUC-ROC ≥ 0.90 и обеспечить интерпретируемость результатов.

### Краткие описания аналогов

#### Аналог 1: Ансамбли бустинга (XGBoost, CatBoost, LightGBM)

Алгоритмы градиентного бустинга деревьев (GBDT) представляют семейство методов, итеративно строящих ансамбли деревьев решений. Реализации XGBoost, CatBoost и LightGBM оптимизированы для работы со структурированными табличными данными. Ключевыми преимуществами является встроенная обработка категориальных переменных (особенно в CatBoost), пропущенных значений и дисбаланса классов. Исследования показывают достижение AUC-ROC в диапазоне 0.85–0.97 для предсказания кардиоваскулярных заболеваний. Основные недостатки связаны с необходимостью подбора гиперпараметров и относительно низкой встроенной интерпретируемостью без применения техник типа SHAP [1, 2, 3].

#### Аналог 2: Полносвязные нейронные сети (DNN/MLP)

Архитектуры с функциями активации ReLU, слоями Dropout и Batch Normalization позволяют моделировать сложные нелинейные взаимодействия между признаками. Достижимы высокие показатели точности (AUC-ROC 0.88–0.94) при правильной настройке параметров. Однако модели требуют большие объёмы данных для обучения, чувствительны к масштабированию входных признаков и имеют проблему интерпретируемости без применения внешних методов [3, 4, 5].

#### Аналог 3: Рекуррентные нейронные сети (LSTM)

LSTM-архитектуры способны моделировать временные последовательности и долгосрочные зависимости. При наличии временной информации в данных (последовательности визитов, история болезни) показывают F1-score 0.88–0.91 для мультибольезневого прогнозирования. Главный недостаток — в статическом сценарии прогнозирования, где временная структура отсутствует, преимущества LSTM минимальны, а вычислительная сложность выше, чем у альтернативных методов [6, 7].

#### Аналог 4: Гибридные ансамбли с методами интерпретируемости (GBDT + SHAP/LIME)

Комбинированный подход, объединяющий высокоточные ансамбли (XGBoost, CatBoost, Random Forest) с техниками объяснения SHAP и LIME. SHAP вычисляет вклад каждого признака на основе теории кооперативных игр, обеспечивая глобальные и локальные объяснения. Сохраняет высокую точность (AUC-ROC 0.85–0.97) и добавляет критическую интерпретируемость для клиничес кой практики. Недостаток — вычислительная сложность SHAP на больших датасетах [8, 9, 10].

#### Аналог 5: Трансформер-подобные архитектуры (SAINT/TabTransformer)

Адаптируют механизм самовнимания из трансформеров для табличных данных. SAINT применяет двухуровневое внимание (внутри признаков и между образцами), TabTransformer объединяет embedding-слои с трансформерной архитектурой. Показывают конкурентоспособность с GBDT (0.85–0.92) и обеспечивают встроенную интерпретируемость через веса внимания. Недостатки: требуют больше данных, на практике часто уступают GBDT на среднего размера датасетах, требуют GPU [11, 12].

### Критерии сравнения аналогов

Выбор трёх ключевых критериев обусловлен необходимостью преодоления фундаментальных вызовов дипломного исследования. Разработанная архитектура должна обеспечить:

1. Высокую предсказательную способность и достижение целевого показателя AUC-ROC ≥ 0.90
2. Обработку реальных медицинских данных с дисбалансом классов (~70% здоровых, ~30% больных), пропущенными значениями и смешанными типами признаков
3. Интерпретируемость результатов для врачей

Критерий 1: Прогностическая точность (AUC-ROC).* Площадь под ROC-кривой (Area Under the ROC Curve) является стандартом медицинского машинного обучения для оценки классификации при дисбалансе классов. Целевое значение ≥ 0.90. Методика измерения: максимальное достигнутое AUC-ROC из рецензируемых публикаций с использованием 5-fold или 10-fold cross-validation.

Критерий 2: Встроенная обработка табличных медицинских данных. Оценивает, как хорошо алгоритм справляется с категориальными переменными, пропущенными значениями и дисбалансом классов без обширной предварительной обработки. Измеряется описанием встроенных механизмов: какие стратегии используются для категориальных переменных, как обрабатываются NaN-значения, какие встроенные техники применяются к дисбалансу.

Критерий 3: Интерпретируемость и поддержка объяснения прогнозов. Измеряет способность алгоритма предоставить объяснения решений. Включает встроенную интерпретируемость (способность модели напрямую объяснить решение), совместимость с методами объяснения (SHAP, LIME), типы возможных объяснений (глобальные и локальные).

### Выводы по итогам сравнения

Проведённый анализ выявил следующие ключевые выводы:

1. GBDT демонстрируют наиболее сбалансированный профиль для задачи. Они последовательно достигают целевой AUC-ROC (0.90–0.97), имеют встроенную обработку всех типов медицинских данных и требуют минимум предварительной обработки.

2. DNN достигают сравнимую точность (0.88–0.94 но требуют значительно больше подготовки: кодирование признаков, явное решение дисбаланса (SMOTE, class weights), тщательная регуляризация. Привлекательны при наличии времени на подбор гиперпараметров.

3. LSTM показывают более низкую точность при отсутствии временной структуры (как в представленном датасете). На статических данных теряют преимущество перед DNN и GBDT при аналогичных требованиях по памяти.

4. Гибридные ансамбли GBDT + SHAP/LIME сохраняют точность базовой модели и добавляют интерпретируемость, критичную для клинических приложений. Внешние валидации показывают 0.76 c-index на новых данных.

5. Трансформер-подобные архитектуры — перспективный, но менее зрелый подход. На среднего размера датасетах часто уступают GBDT, требуют GPU, требуют больше данных и исследований.

Заключение обзора: Для текущей задачи рекомендуется дуальный подход — основной трек DNN (в соответствии с требованием спроектировать архитектуру нейронной сети) с параллельным обучением GBDT + SHAP как бенчмарка и инструмента объяснения.

## Выбор метода решения

На основе сравнительного анализа была выбрана гибридная архитектура, объединяющая глубокую нейронную сеть с методами обработки табличных данных и техниками интерпретируемости SHAP.

### Архитектура решения

Решение реализуется как Python-приложение (модуль) с тремя основными компонентами:

#### 1. Входной модуль обработки данных
- Нормализация числовых признаков (StandardScaler)
- Кодирование категориальных переменных (label encoding)
- Обработка пропущенных значений (mean imputation для числовых, mode для категориальных)
- Обработка дисбаланса классов (SMOTE с sampling_strategy=0.7 на обучающей выборке)

#### 2. Нейросетевой классификатор (MLP)
```
Input(40) → Dense(128, ReLU) → BatchNorm → Dropout(0.3)
          → Dense(64, ReLU) → BatchNorm → Dropout(0.3)
          → Dense(32, ReLU) → Dropout(0.3)
          → Dense(1, Sigmoid)
```
- Входной слой: размер = количество признаков (~40)
- Скрытые слои: 128 → 64 → 32 нейронов с ReLU активацией
- Регуляризация: Dropout (0.3) и Batch Normalization
- Выходной слой: 1 нейрон с сигмоидной активацией (вероятность болезни)
- Функция потерь: Binary Crossentropy с class_weights
- Оптимизатор: Adam (learning_rate=0.001)

#### 3. Модуль интерпретируемости
- Применение SHAP для получения глобальных объяснений (средний вклад каждого признака)
- Локальные объяснения для конкретного пациента (какие факторы увеличили/уменьшили риск)
- Визуализация: бимоде-графики (feature importance), водопадные диаграммы

### Качества решения

1. Точность: AUC-ROC ≥ 0.90 на валидационном наборе (15% данных), целевой AUC-ROC ≥ 0.88 на тестовом наборе

2. Обработка данных: Автоматическая обработка пропущенных значений, категориальных переменных, дисбаланса классов через SMOTE

3. Вычислительная эффективность: Время обучения < 5 минут на GPU (< 30 минут на CPU), время вывода < 100 мс на один пациент

4. Интерпретируемость: Объяснение каждого прогноза в терминах клинических факторов с указанием направления влияния (повышает/понижает риск) через SHAP-значения

5. Воспроизводимость: Фиксированный random seed, полная документация

### Методология валидации

Разделение данных: 70% обучающая, 15% валидационная, 15% тестовая

Обучение модели:
- Early stopping при отсутствии улучшения валидационной loss в течение 10 эпох
- Количество эпох: 100–200
- Мониторинг метрик: AUC-ROC, Accuracy, Precision, Recall, F1 на валидационном наборе
- Сохранение модели с превосходным AUC-ROC

Интерпретация результатов:
- SHAP-объяснитель обучается на подвыборке данных
- Вычисление SHAP-значений для всех тестовых примеров
- Создание визуализаций: глобальная важность признаков, водопадные диаграммы

Альтернативный подход для сравнения: Параллельно обучить CatBoost на сырых данных (встроенная обработка дисбаланса, ~1–2 минуты обучения, обычно AUC-ROC 0.92–0.95)

## Описание метода решения

### Архитектурные принципы

1. Разделение входных данных (CTDE-подобный подход)
Система разделяет входные данные на обучающую, валидационную и тестовую выборки со стратификацией по целевой переменной. Это критично для корректной оценки обобщающей способности на данных с дисбалансом классов.

2. Встроенная обработка дисбаланса через SMOTE
SMOTE применяется только на обучающей выборке для синтеза примеров меньшинства с параметром sampling_strategy=0.7. Это позволяет сбалансировать классы без утечки информации из валидационного и тестового наборов.

3. Нормализация признаков
StandardScaler нормализует числовые признаки к среднему 0 и дисперсии 1, что критично для сходимости DNN. Это предотвращает доминирование признаков с большим диапазоном значений.

4. Архитектура с регуляризацией
Трёхслойная архитектура (128 → 64 → 32 нейронов) с чередованием Dropout и Batch Normalization предотвращает переобучение и стабилизирует обучение. Dropout удаляет случайные нейроны во время обучения, а Batch Normalization нормализует активации каждого слоя.

5. Интерпретируемость через SHAP
SHAP-значения обеспечивают модель-агностичное объяснение предсказаний на основе теории кооперативных игр, показывая вклад каждого признака в отклонение предсказания от среднего.

### Ожидаемые результаты

На основе анализа аналогов ожидается достижение:
- AUC-ROC ≥ 0.90 на валидационном наборе
- Чувствительность (recall) ≥ 0.85 для класса болезни
- Специфичность ≥ 0.85 для класса здоровых
- Время обучения на GPU < 5 минут
- Интерпретируемые SHAP-значения для всех прогнозов

## Заключение

На основе проведённого исследования получены следующие результаты по каждой из поставленных задач:

По задаче 1: Был проведён сравнительный анализ пяти основных парадигм машинного обучения (ансамбли бустинга, полносвязные сети, рекуррентные сети, гибридные системы, трансформеры) для прогнозирования риска заболеваний на табличных данных. Анализ показал, что для статических клинических данных с дисбалансом классов ансамбли GBDT демонстрируют подходящий профиль по всем трём критериям, однако DNN выбраны в качестве основного подхода в соответствии с требованием спроектировать архитектуру нейронной сети.

По задаче 2: Был проведён содержательный сравнительный анализ всех пяти аналогов по трём объективным критериям: прогностическая точность, встроенная обработка табличных медицинских данных и интерпретируемость. Результаты сравнения показали, что гибридные ансамбли с методами объяснения (GBDT + SHAP/LIME) обеспечивают подходящий баланс между точностью и интерпретируемостью.

По задаче 3: На основе результатов сравнительного анализа была обоснована архитектура гибридной системы, объединяющей трёхслойную полносвязную нейронную сеть (128 → 64 → 32 нейронов) с модулями предварительной обработки данных (SMOTE, нормализация, кодирование) и системой объяснения результатов через SHAP-значения. Предложенная архитектура обеспечивает как высокую предсказательную способность (целевой AUC-ROC ≥ 0.90), так и клиническую интерпретируемость.

По задаче 4: Запланирован ряд экспериментов для оценки качества работы архитектуры, включая сравнение различных конфигураций нейронной сети, методов обработки дисбаланса, анализ интерпретируемости через SHAP и валидацию на подвыборках данных по демографическим характеристикам.
