# Сравнение аналогов

## Принцип отбора аналогов

Процесс отбора аналогов был реализован через поиск по специализированным системам научных публикаций (Google Scholar, PubMed, arXiv) и платформе GitHub для нахождения open source решений. Методология фокусировалась на релевантности к задачам прогнозирования риска заболеваний на основе клинических данных пациентов с использованием методов машинного обучения и глубокого обучения.

Были использованы следующие целевые поисковые запросы: "neural network disease risk prediction", "deep learning cardiovascular disease classification", "XGBoost CatBoost medical diagnosis tabular data", "LSTM RNN healthcare prediction", "gradient boosting disease risk", "class imbalance medical prediction SMOTE", "explainable AI SHAP LIME healthcare".

В конечном итоге были выбраны пять аналогов, представляющих основные парадигмы современного подхода к прогнозированию риска заболеваний: (1) ансамбли алгоритмов бустинга (XGBoost, CatBoost), (2) глубокие нейронные сети общего назначения (DNN), (3) рекуррентные нейронные сети (LSTM), (4) гибридные ансамбли с техниками интерпретируемости (GBDT + SHAP/LIME), и (5) трансформер-подобные архитектуры для табличных данных (SAINT/TabTransformer). Выбор этих аналогов обусловлен необходимостью анализа различных подходов к обработке многомерных клинических данных, их способности к достижению целевого показателя AUC-ROC ≥ 0.90, а также к обеспечению интерпретируемости результатов.

## Краткие описания аналогов

### Аналог №1: Ансамбль алгоритмов бустинга (XGBoost, CatBoost, LightGBM)

**Описание**: Алгоритмы градиентного бустинга деревьев (Gradient Boosting Decision Trees, GBDT) представляют собой семейство методов, которые итеративно строят ансамбли деревьев решений для минимизации функции потерь. XGBoost (eXtreme Gradient Boosting), CatBoost (Categorical Boosting) и LightGBM (Light Gradient Boosting Machine) — это оптимизированные реализации, специально разработанные для работы со структурированными табличными данными. Эти алгоритмы автоматически обрабатывают категориальные признаки, поддерживают обработку пропущенных значений и демонстрируют высокую эффективность при работе с клинической информацией о пациентах.

**Ключевые характеристики**:
- Тип: Ансамбль деревьев решений (итеративный бустинг)
- Данные: Работают как с категориальными, так и с числовыми признаками
- Встроенная обработка: Пропущенные значения, дисбаланс классов (class weights)
- Масштабируемость: Параллелизируются между признаками и образцами

**Преимущества**:
- Высокая предсказательная точность на табличных данных медицинского профиля. Исследования показывают достижение AUC-ROC в диапазоне 0.85–0.97 для предсказания кардиоваскулярных заболеваний.
- Встроенная обработка категориальных переменных (особенно важно для диагностических кодов и типов тестов) без необходимости предварительного кодирования.
- Естественная способность к обработке пропущенных данных, что критично для реальных электронных медицинских записей.
- Интерпретируемость через анализ важности признаков и значения Shapley (SHAP).
- Быстрое обучение и вывод даже на больших датасетах с тысячами признаков.

**Недостатки**:
- Требуют тщательного подбора гиперпараметров (learning rate, tree depth, regularization) для избежания переобучения.
- Менее интерпретируемы в сравнении с линейными моделями на уровне отдельных предсказаний без применения техник типа SHAP.
- В случае сильного дисбаланса классов (как в представленном датасете ~70% здоровых, ~30% больных) требуют дополнительной обработки данных (SMOTE, class weights).

**Источники**: [1][2][3]

### Аналог №2: Глубокие нейронные сети (DNN/MLP)

**Описание**: Полносвязные многослойные персептроны (Multilayer Perceptron, MLP) с функциями активации ReLU, включающие слои Dropout для регуляризации и Batch Normalization для стабилизации обучения. Архитектура состоит из входного слоя (размер равен числу признаков), нескольких скрытых слоёв (обычно 2–4 слоя с 64–512 нейронов) и выходного слоя с сигмоидной активацией для бинарной классификации. Такие модели обучаются методом обратного распространения ошибки с оптимизаторами типа Adam.

**Ключевые характеристики**:
- Тип: Полносвязная нейронная сеть (прямое распространение сигнала)
- Данные: Требуют числовых представлений всех признаков (числовые, категориальные кодируются)
- Архитектура: Типично 2–4 скрытых слоя, ReLU активация, Dropout (0.2–0.5)
- Обучение: Adam optimizer, Binary crossentropy loss для бинарной классификации

**Преимущества**:
- Универсальность: могут моделировать сложные нелинейные взаимодействия между признаками.
- Гибкость архитектуры: легко добавлять дополнительные слои, применять различные техники регуляризации.
- Способность к обучению многомодальных данных (если расширить с ветвями для обработки изображений/текста).
- Достижимы высокие показатели точности (AUC-ROC 0.88–0.94) при правильной настройке гиперпараметров.

**Недостатки**:
- "Чёрный ящик": низкая интерпретируемость без применения техник объяснения (LIME, SHAP).
- Требуют больших объёмов данных для хорошей генерализации (обычно тысячи примеров на класс).
- Чувствительны к масштабированию входных признаков и требуют нормализации.
- Долгое время обучения (минуты–часы) в сравнении с алгоритмами бустинга.
- Восприимчивы к переобучению при недостаточной регуляризации.

**Источники**: [3][4][5]

### Аналог №3: Рекуррентные нейронные сети (LSTM)

**Описание**: Long Short-Term Memory (LSTM) — это тип рекуррентной нейронной сети, специально разработанный для обработки последовательных данных и способный запоминать долгосрочные зависимости. При применении к клиническим данным может использоваться для моделирования временных последовательностей визитов пациентов, результатов тестов и прогрессирования симптомов. Архитектура включает ячейки памяти (memory cells) с вентилями (gates) для контроля информационного потока, что позволяет модели избегать проблемы затухающего градиента.

**Ключевые характеристики**:
- Тип: Рекуррентная нейронная сеть (временные последовательности)
- Данные: Требуют временной упорядоченности информации о пациенте
- Архитектура: LSTM ячейки (64–256 единиц), часто с Dropout, выходной слой для классификации
- Особенность: Способны обрабатывать переменные длины последовательностей

**Преимущества**:
- Естественное моделирование временных паттернов в медицинских данных (история болезни, прогресс состояния).
- Высокая точность при наличии временной информации. Исследования показывают F1-score 0.88–0.91 для мульти-болезневого прогнозирования.
- Способность к автоматическому выявлению долгосрочных взаимодействий между признаками.
- Обрабатывают последовательности переменной длины без необходимости предварительной трансформации.

**Недостатки**:
- Требуют временной информации, которая не всегда доступна или релевантна для статического прогнозирования на основе одного стопроцентного среза данных.
- Высокая вычислительная сложность обучения (дольше, чем DNN или GBDT).
- Сложность интерпретации: еще менее прозрачны, чем обычные DNN, так как скрытые состояния LSTM трудно визуализировать.
- При отсутствии явной временной структуры в данных (как в представленном датасете) преимущества LSTM минимальны.

**Источники**: [6][7]

### Аналог №4: Гибридные ансамбли с методами интерпретируемости (GBDT + SHAP/LIME)

**Описание**: Комбинированный подход, объединяющий высокоточные ансамбли методов бустинга (XGBoost, CatBoost, Random Forest) с техниками объяснения искусственного интеллекта — SHAP (SHapley Additive exPlanations) и LIME (Local Interpretable Model-agnostic Explanations). SHAP вычисляет вклад каждого признака в прогноз, основываясь на теории кооперативных игр, позволяя получить глобальные и локальные объяснения. LIME аппроксимирует предсказание чёрного ящика линейной моделью в окрестности примера, обеспечивая локальную интерпретируемость.

**Ключевые характеристики**:
- Тип: Ансамбль + post-hoc объяснение
- Интерпретируемость: Глобальная (средняя важность признаков) и локальная (объяснение отдельного прогноза)
- Методика: SHAP вычисляется через Shapley value, LIME через локальную линейную аппроксимацию
- Вывод: Визуальные графики (бимоде-графики, водопадные диаграммы)

**Преимущества**:
- Сохраняет высокую точность предсказания исходного ансамбля (AUC-ROC 0.85–0.97).
- Обеспечивает медицинскую интерпретируемость: врач может понять, какие признаки (возраст, давление, холестерин) и как влияют на прогноз.
- Ускоряет принятие решений в клинической практике, т.к. врач видит ясное обоснование рекомендации модели.
- Помогает выявить ошибки модели и её смещения (bias), что критично для медицинских приложений.
- SHAP основан на теоретически строгом математическом фундаменте (теория кооперативных игр).

**Недостатки**:
- SHAP вычислительно сложен для больших датасетов и моделей (может требовать часы на большом наборе данных).
- Требует дополнительного этапа обработки после обучения основной модели.
- LIME чувствительна к выбору окрестности и ядра, может быть нестабильной для редких примеров.
- Не избавляет от необходимости правильного выбора исходной модели и обработки данных.

**Источники**: [8][9][10]

### Аналог №5: Трансформер-подобные архитектуры для табличных данных (SAINT/TabTransformer)

**Описание**: Недавно разработанные архитектуры, адаптирующие механизм самовнимания (self-attention) из трансформеров для обработки табличных медицинских данных. SAINT (Self-Attention and Intersample Attention Transformer) применяет двухуровневое внимание: внутри признаков (self-attention) и между образцами (intersample attention). TabTransformer объединяет преобразования категориальных переменных через embedding слои с трансформером. Эти архитектуры потенциально способны лучше улавливать сложные взаимодействия между признаками в сравнении с классическими DNN.

**Ключевые характеристики**:
- Тип: Трансформер-подобная нейронная сеть
- Механизм: Self-attention для взвешивания важности признаков, intersample attention для связи между пациентами
- Данные: Смешанные типы (категориальные через embedding, числовые)
- Масштабируемость: Параллелизируются хорошо, но требуют больше памяти

**Преимущества**:
- Перспективная архитектура: показывает конкурентоспособные результаты с GBDT на медицинских данных.
- Встроенный механизм внимания обеспечивает некоторую интерпретируемость (веса внимания показывают, на какие признаки модель "смотрит").
- Гибкость: легко расширяется на мультимодальные данные (табличные + изображения + текст).
- Меньше требует инженерии признаков благодаря встроенным слоям embedding.

**Недостатки**:
- Относительно новый подход, меньше исследований и примеров применения в медицине.
- Требуют большего объёма данных для хорошей обучаемости в сравнении с GBDT.
- Вычислительно дороже при обучении (требуют GPU).
- До сих пор не достигают точность GBDT на многих табличных медицинских датасетах, особенно на малых выборках.
- Требуют дополнительной настройки и понимания механизма внимания.

**Источники**: [11][12]

## Критерии сравнения аналогов

Выбор трёх ключевых критериев для сравнительного анализа обусловлен необходимостью преодоления фундаментальных вызовов, лежащих в основе проблемы дипломного исследования.

Разработанная архитектура должна обеспечить: 
1. Высокую предсказательную способность и достижение целевого показателя AUC-ROC ≥ 0.90, позволяющего использовать модель в качестве вспомогательного инструмента поддержки клинических решений.
2. Обработку реальных медицинских данных с несбалансированными классами (~70% здоровых, ~30% больных), пропущенными значениями и смешанными типами признаков (категориальные, числовые).
3. Интерпретируемость результатов прогнозирования, позволяющую врачам понять, какие клинические факторы влияют на оценку риска.

### Критерий 1: Прогностическая точность и способность достижения целевого показателя AUC-ROC

**Описание и обоснование**: Критерий оценивает предсказательную эффективность алгоритма, измеряемую площадью под кривой ROC (Area Under the ROC Curve). Этот метрический показатель является стандартом в медицинском машинном обучении для оценки качества классификации при наличии дисбаланса классов, так как ROC-AUC инвариантна к пороговому значению и показывает способность модели различать здоровых и больных пациентов при варьировании чувствительности и специфичности.

**Измеряемая метрика**:
- **AUC-ROC**: максимальное значение, достигаемое на валидационном наборе данных при оптимальной конфигурации гиперпараметров. Целевое значение ≥ 0.90. Источник информации: опубликованные исследования и результаты экспериментов.
- **Методика измерения**: Для каждого аналога указано максимальное достигнутое значение AUC-ROC, зафиксированное в рецензируемых публикациях или документации проектов на GitHub. Используется 5-fold или 10-fold cross-validation при доступности.

---

### Критерий 2: Встроенная обработка табличных медицинских данных (категориальные переменные, пропущенные значения, дисбаланс классов)

**Описание и обоснование**: Критерий оценивает, как хорошо алгоритм справляется с особенностями реальных медицинских датасетов без необходимости обширной предварительной обработки. Эффективная встроенная обработка снижает затраты на инженерию признаков, минимизирует риск утечки информации при обработке данных и позволяет быстрее перейти к разработке прототипа. Это критично, так как представленный датасет содержит смешанные типы переменных и дисбаланс классов.

**Измеряемая метрика** (качественная, но с конкретными показателями):
- **Категориальные переменные**: Способность алгоритма работать с категориальными признаками напрямую, без необходимости one-hot encoding или label encoding. Оценка — перечисление встроенных механизмов (например, "CatBoost использует встроенное категориальное расщепление с сохранением порядка для упорядоченных категорий").
- **Пропущенные значения**: Встроенная обработка NaN-значений. Оценка — описание подхода (например, "XGBoost автоматически выбирает оптимальное направление для пропущенных значений в каждом сплите дерева").
- **Дисбаланс классов**: Встроенные механизмы для работы с несбалансированными данными. Оценка — перечисление методов (class weights, focal loss, встроенная SMOTE и т.д.).

---

### Критерий 3: Интерпретируемость и поддержка объяснения прогнозов

**Описание и обоснование**: Критерий измеряет способность алгоритма предоставлять объяснения своих решений, что критично для медицинских приложений. Врач или регулятор должны понимать, на основе каких факторов модель классифицировала пациента как группу риска. Интерпретируемость также помогает выявить смещения (bias) в данных и модели, убедиться в клинической обоснованности результатов.

**Измеряемая метрика** (качественная с конкретными характеристиками):
- **Встроенная интерпретируемость**: Способность модели напрямую предоставить объяснение (без post-hoc техник). Оценка — описание встроенных методов (например, "DNN: черный ящик без встроенной интерпретируемости; GBDT: встроенная важность признаков через gain/split").
- **Совместимость с методами объяснения**: Поддержка post-hoc техник типа SHAP, LIME. Оценка — указание на возможность применения техник (например, "совместим со всеми SHAP методами", "LIME может использоваться, но требует дополнительных вычислений").
- **Тип объяснений**: Какие типы объяснений возможны (глобальные — средняя важность признаков для всей модели; локальные — объяснение для конкретного пациента). Оценка — перечисление видов объяснений.

## Выводы по итогам сравнения

Проведённый сравнительный анализ пяти аналогов выявил значительные различия в их пригодности для решения задачи прогнозирования риска заболеваний на представленном датасете (100,000 пациентов, 40 признаков, дисбаланс ~70%/30%).

**Ключевые наблюдения**:

1. **XGBoost/CatBoost/LightGBM (GBDT) демонстрируют наиболее сбалансированный профиль**. Эти алгоритмы последовательно достигают высокие показатели AUC-ROC (0.90–0.97), имеют встроенную обработку категориальных переменных (особенно CatBoost), пропущенных значений и дисбаланса классов. На практике GBDT требуют минимум предварительной обработки данных и быстро сходятся. CatBoost показывает AUC-ROC до 0.97 на данных хронических заболеваний. XGBoost достигает 0.95+ при работе с психологическими факторами. Эти свойства критичны при работе с реальными медицинскими данными.

2. **Глубокие нейронные сети (DNN) достигают сравнимую точность (0.88–0.94), но требуют значительно больше подготовки данных**. DNN могут достичь AUC-ROC 0.98 при правильной обработке (SMOTE для дисбаланса, нормализация признаков), но требуют:
   - Предварительного кодирования всех категориальных переменных
   - Явного решения проблемы дисбаланса (SMOTE, class weights в loss)
   - Тщательной регуляризации (Dropout, Batch Norm) для избежания переобучения
   - Больше времени на обучение
   
   DNN становятся привлекательными при наличии достаточных вычислительных ресурсов и времени на подбор гиперпараметров.

3. **LSTM (рекуррентные сети) показывают более низкую точность при отсутствии явной временной структуры данных**. LSTM эффективны при наличии последовательностей визитов или истории болезни (AUC-ROC 0.88–0.91 с временными данными). Однако для статического прогнозирования на основе одного снимка клинического состояния LSTM теряют преимущество перед DNN и GBDT и требуют больше вычислений. На представленном датасете, где нет явной временной информации, использование LSTM нецелесообразно.

4. **Гибридные ансамбли GBDT + SHAP/LIME (Аналог 4) сохраняют точность базовой модели и добавляют критическую интерпретируемость**. Этот подход унаследует преимущества GBDT (высокая точность 0.90–0.95) и при этом обеспечит врачам понимание решений модели через SHAP-значения (глобальные важности и локальные объяснения) и LIME. Внешние валидации показывают, что такие системы достигают 0.76 c-index на новых данных (NPC датасет). Недостаток — вычислительная сложность SHAP на больших датасетах (хотя для 100K пациентов всё ещё выполнимо за разумное время).

5. **Трансформер-подобные архитектуры (SAINT/TabTransformer) — перспективный, но менее зрелый подход**. Эти модели показывают конкурентоспособность с GBDT (0.85–0.92), имеют встроенный механизм внимания для интерпретируемости, но:
   - Требуют больше данных для обучения
   - На практике часто уступают GBDT на среднего размера датасетах (<1 млн примеров)
   - Требуют GPU для эффективного обучения
   - Меньше исследований по применению в медицине
   
   Рекомендуются при наличии особых требований (мультимодальные данные) или при желании использовать последние архитектуры.

### Соответствие задаче дипломного исследования:

- **Целевой показатель AUC-ROC ≥ 0.90**: Все аналоги способны достичь целевой показатель, но GBDT (Аналог 1) и DNN (Аналог 2) наиболее надёжны.
- **Обработка табличных медицинских данных с дисбалансом**: GBDT (особенно CatBoost) требуют минимум подготовки. DNN требуют больше обработки данных.
- **Архитектура нейронной сети**: Явно требуется для текущего семестра (спроектировать архитектуру). DNN и Трансформеры соответствуют этому требованию. GBDT — не нейронные сети, но показывают лучшую производительность на табличных данных.
- **Интерпретируемость для валидации**: Гибридный подход (Аналог 4: GBDT + SHAP/LIME) обеспечивает как точность, так и объяснения.

### Выбор оптимального направления:

На основе анализа, для текущего семестра рекомендуется **дуальный подход**:
1. **Основной трек**: Разработать архитектуру глубокой нейронной сети (DNN с 2–3 скрытыми слоями, ReLU, Dropout, Batch Norm), способную достичь AUC-ROC ≥ 0.90. Это соответствует требованию спроектировать архитектуру нейронной сети.
2. **Параллельный трек для сравнения**: Обучить ансамбль GBDT (XGBoost + CatBoost) и применить SHAP для интерпретации. Это служит как бенчмарк и как инструмент объяснения для валидации.

Этот подход позволит достичь целевые метрики, получить опыт архитектуры нейронных сетей и одновременно обеспечить интерпретируемость для клинической валидации модели.

## Выбор метода решения

### Обоснование выбора архитектуры:

На основе сравнительного анализа аналогов было принято решение о разработке **гибридной системы прогнозирования**, объединяющей глубокую нейронную сеть (DNN) с методами обработки табличных данных и техниками интерпретируемости.

**Архитектура решения должна представлять собой**:

Python-приложение (модуль/скрипт), реализующее:
1. **Входной модуль обработки данных**: Нормализация числовых признаков (StandardScaler), кодирование категориальных переменных (label encoding или one-hot в зависимости от типа признака), обработка пропущенных значений (mean imputation для числовых, mode для категориальных), обработка дисбаланса классов (SMOTE на обучающей выборке).

2. **Нейросетевой классификатор**: Полносвязная архитектура MLP с:
   - **Входной слой**: размер = количество признаков (~40)
   - **Скрытые слои**: 3 слоя размерности 128 → 64 → 32 нейронов с ReLU активацией
   - **Регуляризация**: Dropout (rate=0.3) после каждого скрытого слоя, Batch Normalization для стабилизации
   - **Выходной слой**: 1 нейрон с сигмоидной активацией для бинарной классификации (вероятность болезни)
   - **Функция потерь**: Binary Crossentropy с class_weights для учёта дисбаланса
   - **Оптимизатор**: Adam (learning rate=0.001)

3. **Модуль интерпретируемости**: Применение SHAP для получения:
   - **Глобальные объяснения**: значения SHAP для каждого признака (средний вклад в прогноз)
   - **Локальные объяснения**: для конкретного пациента — какие признаки увеличили/уменьшили риск
   - **Визуализация**: бимодовые графики (feature importance), водопадные диаграммы (для отдельного пациента)

### Качества, которыми должно обладать решение:

1. **Точность**: Достижение AUC-ROC ≥ 0.90 на валидационном наборе (15% данных) и тестовом наборе (15% данных), с целью AUC-ROC ≥ 0.88 на независимом тестовом наборе.

2. **Обработка данных**: 
   - Автоматическая обработка пропущенных значений
   - Кодирование категориальных переменных
   - Обработка дисбаланса классов (SMOTE с параметром sampling_strategy=0.7 для синтеза 70% от размера большинства)
   - Масштабирование признаков (StandardScaler)

3. **Вычислительная эффективность**: 
   - Время обучения < 5 минут на GPU (или < 30 минут на CPU) для всего датасета
   - Время вывода < 100 мс для одного пациента

4. **Интерпретируемость**:
   - Возможность объяснить каждый прогноз врачу в терминах клинических факторов
   - SHAP-значения показывают направление влияния (повышает/понижает риск) и величину влияния

5. **Воспроизводимость**: 
   - Фиксированный random seed для сравнимости результатов
   - Полная документация по обработке данных и архитектуре

### Способ реализации решения (на основе выводов из обзора):

**Архитектура обучения и валидации**:

1. **Разделение данных**: 70% обучающая выборка, 15% валидационная, 15% тестовая

2. **Обработка обучающей выборки**:
   - Кодирование категориальных признаков (label encoding или one-hot)
   - StandardScaler для нормализации числовых признаков
   - SMOTE для балансировки (только на обучающей выборке!)
   - Результат: Xtr_processed, ytr_balanced

3. **Обучение модели**:
   - Архитектура: Input(40) → Dense(128, ReLU) → Batch Norm → Dropout(0.3) → Dense(64, ReLU) → Batch Norm → Dropout(0.3) → Dense(32, ReLU) → Dropout(0.3) → Dense(1, Sigmoid)
   - Функция потерь: Binary Crossentropy с class_weights (вес для класса 1 = количество класса 0 / количество класса 1)
   - Оптимизатор: Adam(lr=0.001)
   - Early stopping: мониторить валидационную loss, остановка при отсутствии улучшения в течение 10 эпох
   - Количество эпох: 100–200

4. **Валидация**: На каждой эпохе вычислять AUC-ROC, точность, полноту, F1 на валидационной выборке. Сохранять модель с наилучшим AUC-ROC.

5. **Тестирование**: На финальной модели вычислить метрики на тестовой выборке (AUC-ROC, Accuracy, Precision, Recall, F1).

6. **Интерпретация**:
   - Обучить SHAP-объяснитель на подвыборке обучающих данных (для ускорения)
   - Вычислить SHAP-значения для всех тестовых примеров
   - Создать графики: бимод (global feature importance), водопадные диаграммы для выбранных пациентов

**Альтернативный подход (рекомендуется для сравнения)**:

Параллельно обучить CatBoost модель на сырых данных (без SMOTE, т.к. встроена обработка дисбаланса):
   - Тогда не требует предварительной обработки категориальных переменных
   - Встроенный scale_pos_weight для дисбаланса
   - Быстрое обучение (~1–2 минуты)
   - Обычно достигает AUC-ROC 0.92–0.95
   - Результаты могут служить бенчмарком

**Экспериментальная оценка**:

Провести следующие эксперименты для оценки качества:

1. **Сравнение архитектур нейронных сетей**: 
   - Базовая архитектура (128 → 64 → 32)
   - Глубокая архитектура (256 → 128 → 64 → 32 → 16)
   - Метрика сравнения: AUC-ROC, время обучения
   - Вывод: выбрать архитектуру с лучшим балансом точности и скорости

2. **Влияние обработки дисбаланса**:
   - Без SMOTE (только class weights)
   - С SMOTE (sampling_strategy=0.7)
   - С SMOTE-TOMEK (комбинированная обработка)
   - Метрика: AUC-ROC, recall для класса болезни (важна чувствительность)
   - Вывод: выбрать подход, максимизирующий чувствительность при сохранении специфичности

3. **Сравнение интерпретируемости**:
   - SHAP-значения для DNN модели
   - Feature importance для CatBoost
   - Метрика: согласованность интерпретаций, клиническая обоснованность
   - Вывод: выявить ключевые клинические факторы, которые влияют на риск

4. **Валидация на подвыборках**:
   - Стратификация по возрасту, полу, типам заболеваний
   - Вычисление AUC-ROC для каждой подвыборки
   - Вывод: убедиться, что модель обобщается и не имеет значительного смещения
